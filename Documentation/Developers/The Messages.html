The Messages

Introduction

JedPoint is built on a microkernel architecture, using messages as the way of communication between modules. The messages are stored in a class called JedPointMessage. The messages themselves contain a request and a response part. The module sending the message fills in the request, and the module receiving the message fills in the response.

Requests and Responses

Reading and writing request and response data is done using the following methods: setRequest, setResponse, getRequest and getResponse.

In case you need to clear a request or response, us clearRequest and clearResponse.

Summary of Message types

Messages are sent to one of the several types of module types. Below is a list of module types which are linked to a list of messages for the module type.

Description

Writes an error message to the log file. An error is an exception that caused the module to abort whatever it was doing at the time. Most of the time it means that the module can still function, but the message it received caused it to throw an exception. If a module is unable to function at all (due to a missing config file or perhaps a server that didn't answer when it should) then a Fatal should be called.

An Area

An area consists of an areaname, a description and tagline and origin strings. The tagline+origin strings can be either strings (in case you want to use only one tagline / origin) or they can point to real files, in which case the tagline + origin generators will notice that and grab a tagline/origin out of the files at random.
